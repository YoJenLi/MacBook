
作者xup6fm06 (無尾熊)看板Tech_Job標題[請益] 請假扣薪算法時間Thu Sep 18 09:36:24 2014
最近遇到一間公司C
請假扣薪的算法是:
 底薪 / 當月實際上班時數 x 請假時數
 EX.34000 / 170H x 8.5 = 1700元
   (假設當月出勤上班時數170,請假當天出勤工時8.5)

全勤每間公司各有規定,我覺得可以接受

但公司C的HR說
這樣的請假扣薪方式, 符合勞基法

但是我以往遇到的公式都是:
 底薪 / 30天 / 8h x 請假時數
 EX.34000 / 30天 / 8 x 8.5 = 1204元

請一天假,前者等於比後者 多扣了4成薪水
(1700-1204)/1204 = 0.41 = 41%

問了當人資的朋友
說勞基法沒有這部分的規定
所以公司C, 可以在勞動契約自己規範

但是自己google,是說應該以 除30天 除8h 計算

請教版上各位板友
有人遇到以 出勤時數 計算的扣法嗎?
這樣是否違反勞基法?
或是勞基法真的在這塊是模糊空間的?
