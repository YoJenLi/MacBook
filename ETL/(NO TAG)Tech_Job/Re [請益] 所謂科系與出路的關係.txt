這就是為什麼說剛畢業的會"稍微"看一下學校和成績單
沒經驗就會看你過去的學習態度和領域
主管從面談中更進一步了解能不能training

如果你連 C code 的 ; 代表什麼意思都不知道，
再有興趣，韌體主管也不敢用你阿
如果你成績不錯，但就是不喜歡進無塵室
主管也會想這樣做的久嗎?
興趣跟專業都會考量

通常大學跟研究所的學分會一起看
大學電機 + 光電所 (做光纖，偏材料領域)
最後有人在HTC寫 code 跟鴻海做RF
這是我朋友的例子

而我也是類似背景，在面板廠待了三年轉IC設計
在面板廠我是畫光罩跟處理光學issue，基本上跟IC設計完全無關
但也是憑大學成績單 (不是研究所) + 興趣 + 他們需要我在面板廠的專業
說服面試我的主管
我這單位在系統廠是RD，因為工作內容跟系統廠RD一模一樣
但對IC設計來說是FAE非RD (規模大的會把跑客戶的歸FAE，偏設計的歸SAE)
這樣講應該有體會到IC設計的RD有一定的門檻

而製程方面的工作
其實比較像做產線的，門檻比較沒有科系上的限制
我覺得大致上可以這樣分
喜歡寫程式兜電路的不喜歡做製程
喜歡做製程的不喜歡寫程式兜電路那種工作
